1、伪类选择器

.link,.visited—–应用于a标签，用于超链接
.focus,.hover,active ——应用于一般元素但是一般元素的 focus 显示不出，在 input 元素上显示较明显。

2、高级选择器

高级选择器有子选择器、相邻同胞选择器、

  • 子选择器

后代选择器选择一个元素的所有后代，子选择器只选择元素的直接后代，

  • 相邻同胞选择器（要点：同胞–同一个父元素、相邻—就是相邻）

要点解释： 为同一个父元素下的某个元素之后并且与该元素是相邻的元素。

特殊应用(其实也不特殊，应用上面的规则进行类比)：

第一个Li 之后的元素会应用样式

原因： 从第一个li开始往下，第一个li与第二个li 是同胞元素，应该给第二个元素应用样式，从第二个元素向下找，第三个元素与第二个元素是相邻同胞元素，应该为第三个元素用用样式。。。类推

  • ~ 一般后代选择器

x~y 匹配任何在 x元素之后的同级p 元素

匹配在 h2 元素之后的所有p 元素

3、属性选择器

  • title属性

在鼠标悬浮在某个元素上的时候 title 元素的信息将会显示出来，仅在鼠标悬浮的时候。

属性选择器的样式选择有很多

4、层叠性和特殊性

问题：层叠性和继承

特殊性：为每种选择器分配一个数字值，将规则的每个选择器的值加在一起，计算出规则的特殊性，特殊性决定优先性。

选择器的特殊性分为4个等级：a,b,c,d

  • 行内样式 a = 1
  • b等于id选择器的总数
  • c等于类、属性、伪类的总数
  • d等于伪元素（::before 、::after）和类型选择器（元素选择器）的数量

样式表的特殊性描述了不同规则的相对权重。

有 id选择器的规则，比没有id选择器的规则特殊

有类选择器的规则，比只有元素选择器的规则特殊

规则的特殊性相同，后声明的优先级较高

出现特殊性冲突—为父元素添加id以提高特殊性

层叠—–覆盖—–讲优先级/特殊性—–应当是是设置的

样式表中的特殊性：

一般性样式应当非常一般

特殊性样式应当非常特殊

要是不得不多次覆盖一般样式，应当显示的应用于需要它的元素。

5、继承

应用样式的元素后代会继承样式的某些属性，如：颜色、字号

一般来说常见的不能被继承的属性：

在特殊性的框架下，被继承的特性值为0，意味着任何显示声明的规则将会覆盖其继承样式

6、引用css 文件到html文件的方法

  • 行内 内嵌 优先级高
  • style
  • link
  • @import

三种方法最终达到的效果相同，但是对于 SEO和网页打开速度有不同的影响。

@import 的方式

推荐的是使用link的方式

link 与import方式的比较

引入css样式的方式不同：

使用link 链接的方式是客户端浏览你的网页时先将外部的css文件加载到网页中，然后再进行编译显示，所以显示出来的网页与预期的一样，网速再慢也是一样的。
@import 导入是客户端在浏览网页的时候，先将Html结构呈现出来，再将外部的css文件加载到网页上，最终的结果是一样的。但是网速很慢的时候会出现没有 css的html网页，会给阅读者不好的体验。
@import 导入比链接要慢
